Implement a Key-Value Store Persister

We persisted Book objects by writing/reading their properties one after the other. This approach is prone to error because you must make sure your code matches the persisted format for both reading and writing. If you change the Book class - for example, adding an edition field to it - then you may no longer be able to read from old files because some lines could be read into the wrong properties. A potential solution is to save the data as key-value pairs. Java has the Properties class, which you can use to store value strings with associated key strings:

public class PropertyBookPersister implements Persister<Book>{
private final static String STORE = "Book.txt";
@Override
public void serialize(Book book) throws IOException {
Properties p = new Properties();
p.setProperty("title", book.getTitle());
p.setProperty("isbn", book.getIsbn());
p.setProperty("publisher", book.getPublisher());
p.setProperty("year", Integer.toString(book.getYear()));
p.setProperty("author", book.getAuthor());
FileWriter fw = new FileWriter(STORE);
p.store(fw, "comment"); // replace "comment" with any relevant comment!
fw.close();
}
@Override
public Course deserialize() throws IOException {
Properties p = new Properties();
FileReader fr = new FileReader(STORE);
p.load(fr);
fr.close();
String title = p.getProperty("title");
String isbn = p.getProperty("isbn");
String publisher = p.getProperty("publisher");
String year = p.getProperty("year");
String author = p.getProperty("author");
return new Book(title, isbn, publisher, year, author);
}
}
